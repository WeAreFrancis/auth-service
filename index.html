  • Response

URI Parameters

  • userId: required (string)

Headers

  • Authorization: required (string)

    The JWT

    Example:

    Bearer my-jwt

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • email: (string)

    The email (only if you're admin or you're the user himself)

  • enabled: (boolean)

    True if the user is enabled, false otherwise (only if you're admin)

  • id: required (string)

    The ID of the user

  • locked: (boolean)

    True if the user is locked, false otherwise (only if you're admin)

  • role: (one of USER, ADMIN, SUPER_ADMIN)

    The role of the user (only if you're admin or you're the user himself)

  • username: required (string)

    The username of the user

Example:

{
  "id": "636c76e7-a75d-4f2f-abd2-82eee12e087f",
  "email": "johndoe@my-domain.com",
  "enabled": true,
  "locked": false,
  "role": "ADMIN",
  "username": "johndoe"
}

HTTP status code 401

Body

Media type: application/json

Type: object

Properties
  • timestamp: required (number)

    The timestamp of the request

  • status: required (number)

    The HTTP error status

  • exception: required (string)

    The class name of the exception that caused the error

  • message: required (string)

    The message of the error

  • path: required (string)

    The URI of the request

Example:

{
  "timestamp": 1494680060971,
  "status": 400,
  "error": "Unauthorized",
  "exception": "com.wearefrancis.auth.exception.BadCredentialsException",
  "message": "Bad credentials",
  "path": "/users/636c76e7-a75d-4f2f-abd2-82eee12e087f/unlock"
}
